# Visual Restraint

## Monochrome-First

- Start with neutral surfaces, gray text hierarchy, and black/white anchors before introducing color.
- Let typography and spacing establish hierarchy; color should reinforce, not replace, that structure.
- A monochrome layout that reads clearly will still read clearly after adding accents; the reverse is rarely true.

## Accent Economy

- Limit semantic accents to one per visual unit (row, card, section).
- When multiple elements compete for attention, readers scan instead of read.
- Choose one signal: a colored border, a status badge, OR colored text—not all three.
- Reserve bright, saturated colors for the single most important signal in a view.
- If two elements both need emphasis, question whether both actually do.

## Visual Weight Distribution

- Not every element deserves equal presence; flatten the hierarchy and nothing stands out.
- Metadata, timestamps, and secondary labels should recede; primary content should anchor the eye.
- When everything looks important, nothing guides the user through the content.
- Test by squinting: the important elements should still be distinguishable.

## Status Indicators

- Status badges and pills should be muted by default; loud colors are for urgent or actionable states only.
- Prefer soft tinted backgrounds over saturated fills for categorical labels.
- A status badge next to status-colored text creates redundancy and noise.
- Section labels that scream (bright colors, large text) steal attention from the rows that carry the real information.
- Let the content rows communicate urgency; section headers should organize, not alarm.

## Saturation and Tone

- Desaturated palettes feel calmer and more editorial; saturated palettes feel more app-like and louder.
- Match saturation to the product tone: a task list is not a game UI.
- When in doubt, reduce saturation by one step and see if clarity improves.
