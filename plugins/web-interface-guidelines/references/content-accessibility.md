# Content and Accessibility

- Prefer inline help over tooltips. Use tooltips only when inline explanation would be excessive.
- Design all states: empty, sparse, dense, success, warning, and error.
- Keep page titles accurate to the current view and task.
- Use redundant status cues. Color alone is not enough; pair it with text or iconography that still communicates meaning non-visually.
- Name icon-only buttons with descriptive accessible labels.
- Prefer native semantics before custom ARIA. Use real buttons, links, labels, headings, and tables whenever possible.
- Keep heading levels ordered and provide a skip link for main content on full pages.
- Ensure decorative elements are hidden from assistive tech and meaningful elements have accessible names.
- Make layouts resilient to short, average, and very long user-generated content.
- Format dates, times, numbers, currencies, and delimiters according to the user's locale.
- Prefer explicit language settings over location inference when deciding locale and language behavior.
- Use non-breaking spaces or other glue characters when units, shortcuts, or product names should stay together.
